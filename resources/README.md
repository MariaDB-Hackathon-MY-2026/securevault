# SecureVault Resources

Supporting references live here. Unlike `docs/`, this directory is not the product handbook for the repo. It exists for external standards, curated links, and implementation guidance that support the codebase.

## What belongs here

- [auth-dashboard-design-system.md](./auth-dashboard-design-system.md): the implemented auth and dashboard UI language, component usage rules, and cleanup backlog
- [development-resources.md](./development-resources.md): curated official docs for the frameworks, services, and libraries used in SecureVault
- [security-standards.md](./security-standards.md): external security standards, OWASP references, and derived handling guidance

## What does not belong here

- Product walkthroughs and repository architecture docs: keep those in [`../docs/`](../docs/)
- API surface documentation: use [`../docs/05-api-reference.md`](../docs/05-api-reference.md) as the single source of truth
- Duplicate summaries of material already covered by the handbook or README
